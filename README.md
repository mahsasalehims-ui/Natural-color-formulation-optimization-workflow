# Natural Color Formulation Optimization Workflow

An agentic pipeline that turns raw spectrophotometer intake data into ranked, actionable formulation specifications for natural food & beverage colorants — built by a plant biologist & food scientist to demonstrate practical Agentic AI applications in R&D formulation workflows.

> 📄 **[Read the full showcase report →](docs/NaturalColor_Agent_Showcase.pdf)**
> Architecture diagrams, scientific rationale, and live pipeline results in one PDF.

## What this does

Given raw spectrophotometer output (reflectance spectrum, CIELAB, XYZ) for incoming natural pigment lots — anthocyanins, carotenoids, phycocyanin — the pipeline runs three autonomous specialist agents in parallel and produces:

| Output | Generated by |
|---|---|
| **Corrective formulation recipe** | Correction Agent — Kubelka-Munk additive mixing model |
| **QC pass/fail certificate** | QC Validator Agent — CIEDE2000 + spectral RMSD vs. tolerance |
| **Shelf-life prediction** | Stability Agent — gradient-boosted regression on pH, water activity, processing temperature |

An orchestrator agent plans which specialist agents to call based on the intake event, merges their outputs into a single ranked candidate list, and routes the result to a human reviewer — whose approval feeds back to retrain the underlying models.

```
data in (spectrophotometer CSV) → 6-step pipeline → ranked candidates + PDF report
```

## Why agentic, not just automated

- **Autonomous tool selection** — a lot that passes QC outright never triggers the correction agent.
- **Bounded self-decision** — each agent evaluates its own gate (under-concentrated? ΔE over tolerance? below shelf-life spec?) and decides its own next action.
- **Tool-calling architecture** — agents call external tools (K-M model, CIEDE2000, trained GBR predictor, knowledge stores) rather than hardcoded inline formulas.
- **Human-in-the-loop feedback** — approvals retrain the system; it improves with use.

## The six pipeline steps

1. **Load & inspect** — read intake lots + reference targets
2. **Normalise & validate** — confirm D65/CIE 1931 2° illuminant, check R(λ) range
3. **Spectral feature extraction** — K/S ratio (Kubelka-Munk), dominant wavelength, ΔE₀₀, spectral RMSD
4. **Corrective addition engine** — solve for corrective pigment dose using K-M additive mixing
5. **Batch QC validation** — CIEDE2000 + RMSD tolerance gates → PASS / MARGINAL / FAIL
6. **Stability model** — train gradient-boosted regression on confirmed shelf-life data, predict for new lots

## Scientific grounding

The pipeline's decision logic is built on published pigment chemistry, not generic heuristics:

- **Anthocyanins** — pH-dependent flavylium cation equilibrium; correction is meaningless above the pigment's stability pH window.
- **Carotenoids** — cis/trans isomerisation as the dominant heat/light degradation pathway.
- **Phycocyanin** — thermal denaturation above ~70°C cannot be corrected by pigment addition; the model distinguishes this from simple dilution by tracking spectral *shape*, not just intensity.
